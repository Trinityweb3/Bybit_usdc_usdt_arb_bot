# Bybit USDT/USDC Perp Spread Bot

Сканирует все линейные перпетуалы на Bybit, находит монеты, у которых есть
одновременно `XXXUSDT` и `XXXUSDC` контракт, и при появлении спреда между
ними открывает лонг на одной ноге и шорт на другой на одинаковое количество
токенов (по ~100 USDT и ~100 USDC ноционала, плечо 10x по умолчанию).

**Бот только открывает позиции.** Никакого автозакрытия нет — нашёл спред,
открыл обе ноги, отправил `/gm` в Telegram-чат и больше этой монетой не
занимается. Управление позицией (закрытие, стоп) остаётся на вас.

## Как это работает

1. `GET /v5/market/instruments-info` — один раз при старте находит все базовые
   монеты, торгуемые и против USDT, и против USDC.
2. Каждые `POLL_INTERVAL_SECS` секунд — `GET /v5/market/tickers` одним
   запросом получает bid/ask по всем символам сразу.
3. Спред считается по исполняемым ценам: `(bid одной ноги − ask другой) / ask`
   в обе стороны.
4. Если спред превышает `ENTRY_SPREAD_PCT` и по этой монете ещё не было входа
   — бот выставляет плечо и одновременно шлёт Market-ордер на покупку
   дешёвой ноги и Market-ордер на продажу дорогой ноги, с одинаковым `qty`.
5. Если одна из двух ног не прошла — бот сразу закрывает вторую ногу
   reduce-only ордером, чтобы не остаться с "голой" позицией (это единственный
   случай, когда бот сам закрывает позицию).
6. После успешного открытия обеих ног — шлёт `/gm` через Telegram Bot API в
   указанный чат и помечает эту монету как "уже отработана", чтобы не
   открывать её повторно на каждом цикле, пока спред держится.

## Запуск

```bash
export BYBIT_API_KEY="..."
export BYBIT_API_SECRET="..."
export TELEGRAM_BOT_TOKEN="123456:AA..."   # токен бота от @BotFather
export TELEGRAM_CHAT_ID="-100123456789"     # id чата/канала, куда слать /gm
export DRY_RUN=true        # сначала обязательно так — только логирует, ничего не отправляет
# export BYBIT_TESTNET=true  # можно погонять на тестнете перед мейннетом

cargo run --release
```

`TELEGRAM_CHAT_ID` бот должен получить заранее — проще всего написать боту
любое сообщение в личку или добавить его в чат, затем прочитать
`chat.id` через `https://api.telegram.org/bot<TOKEN>/getUpdates`.

## Переменные окружения

| Переменная            | По умолчанию | Описание                                             |
|------------------------|--------------|-------------------------------------------------------|
| `BYBIT_API_KEY`         | —            | обязателен                                             |
| `BYBIT_API_SECRET`      | —            | обязателен                                             |
| `TELEGRAM_BOT_TOKEN`    | —            | обязателен                                             |
| `TELEGRAM_CHAT_ID`      | —            | обязателен                                             |
| `BYBIT_TESTNET`         | `false`      | `true` → api-testnet.bybit.com                         |
| `DRY_RUN`               | `true`       | `true` → только логирует, ордера и Telegram не шлёт    |
| `LEVERAGE`              | `10`         | плечо на обеих ногах                                   |
| `NOTIONAL_USDT`         | `100`        | целевой ноционал в USDT-ноге                           |
| `NOTIONAL_USDC`         | `100`        | целевой ноционал в USDC-ноге                           |
| `ENTRY_SPREAD_PCT`      | `0.004`      | 0.4% — порог входа                                     |
| `POLL_INTERVAL_SECS`    | `3`          | частота опроса тикеров                                 |

## Важные ограничения и риски — прочитать перед реальным запуском

- **Позиции не закрываются автоматически.** Бот открывает и забывает —
  мониторить и закрывать нужно вручную (или отдельным скриптом/ботом).
  Держите это в голове при расчёте плеча и ноционала: 10x без плана выхода
  означает, что расхождение цен между ногами может съесть маржу быстрее,
  чем вы успеете среагировать.
- **По каждой монете вход только один раз за сессию.** Список "уже
  отработанных" монет живёт только в памяти процесса — при перезапуске бота
  он про них забудет и может снова войти, даже если позиция ещё открыта.
- **Комиссии и фандинг съедают спред.** Taker-комиссия на обеих ногах — это
  уже ~0.1–0.15% на вход, а фандинг у USDT- и USDC-перпетуалов на одну и ту
  же монету часто разный. `ENTRY_SPREAD_PCT = 0.4%` — отправная точка, а не
  гарантия прибыли.
- **USDC не всегда равен $1.** Спред считается по номиналу без поправки на
  курс USDC/USDT. При депеге USDC это даст ложный сигнал.
- **Нет вебсокета.** REST-поллинг раз в несколько секунд — для гонки за
  быстросходящимися спредами может не хватать скорости; для этого нужен
  `wss://stream.bybit.com/v5/public/linear`.
- **Ордера — Market/IOC**, возможен слиппедж, особенно на тонких стаканах у
  альткоинов.

Обязательно сначала прогоните с `DRY_RUN=true` (и/или `BYBIT_TESTNET=true`),
убедитесь по логам, что бот находит разумные пары и спреды, и только потом
включайте `DRY_RUN=false`.
